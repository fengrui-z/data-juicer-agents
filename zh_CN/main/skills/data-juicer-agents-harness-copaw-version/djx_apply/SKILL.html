Data-Juicer Skills: Apply (Execution Reference)#

Main flow: See the data-juicer skill. This skill provides detailed reference for apply_recipe execution.

Prerequisites#

Condition

Requirement

Verification Command

Plan file

YAML format, exists and is valid

cat plan.yaml

Environment

py-data-juicer>=1.4.0 installed

dj-process --version

Input data

Dataset path is correct and accessible

head -n 1 <dataset_path>

Core Concepts#

Concept

Description

plan_path

YAML plan file path, generated by plan_save

dry_run

Validation mode — does not actually execute; checks plan validity

timeout

Execution timeout in seconds; terminates after timeout

confirm

Execution confirmation — must be true for actual execution

bool

No

Key: --yes (CLI flag) and "confirm": true (JSON field) are both required. --yes skips the interactive prompt; confirm provides logical confirmation.

Command Format#

# Dry-run validation (recommended to run first)
djx tool run apply_recipe --yes --input-json '{"plan_path": "<plan.yaml>", "dry_run": true}'

# Actual execution
djx tool run apply_recipe --yes --input-json '{"plan_path": "<plan.yaml>", "confirm": true, "timeout": 600}'

Retry Strategy#

Attempt

Action

1st failure

Read the error message carefully; check exit code and stderr; fix input and retry

2nd failure

Stop. Report the error. Do not continue retrying.

Maximum 2 apply_recipe attempts. Infinite retries waste budget and are futile.

Must-Read Pitfalls#

1. Both --yes and confirm:true Are Required#

Wrong: Passing only one

# Missing --yes
djx tool run apply_recipe --input-json '{"plan_path": "plan.yaml", "confirm": true}'
# → confirmation_required (exit 3)

# Missing confirm
djx tool run apply_recipe --yes --input-json '{"plan_path": "plan.yaml"}'
# → No response (confirm defaults to false)

Correct: Pass both

djx tool run apply_recipe --yes --input-json '{"plan_path": "plan.yaml", "confirm": true, "timeout": 600}'

2. Increase Timeout for Large Datasets#

Default 300s is insufficient for large datasets:

# 100K+ records
djx tool run apply_recipe --yes --input-json '{"plan_path": "plan.yaml", "confirm": true, "timeout": 1800}'

3. Dry-run Before Execution#

# Step 1: Validate
djx tool run apply_recipe --yes --input-json '{"plan_path": "plan.yaml", "dry_run": true}'

# Step 2: Execute after confirmation
djx tool run apply_recipe --yes --input-json '{"plan_path": "plan.yaml", "confirm": true, "timeout": 600}'

Typical Workflow#

# 1. Validate plan
djx tool run apply_recipe --yes --input-json '{"plan_path": ".djx/plans/my_plan.yaml", "dry_run": true}'

# 2. Execute
djx tool run apply_recipe --yes --input-json '{
  "plan_path": ".djx/plans/my_plan.yaml",
  "confirm": true,
  "timeout": 600
}'

# 3. Verify output
djx tool run inspect_dataset --input-json '{"dataset_path": "<output_path>", "sample_size": 10}'
